# Architecture Decision Records (ADR)

This directory contains records of architectural decisions made for the E-Com Lite project.

## Convention
Each ADR should follow a simple format and be named with a sequential number and short description (e.g., `0001-some-decision.md`).

Format:
- **Title:** Short, descriptive title.
- **Status:** Proposed, Accepted, Rejected, Deprecated.
- **Context:** Why we need to make a decision.
- **Decision:** What we decided to do.
- **Consequences:** Impact of the decision.
