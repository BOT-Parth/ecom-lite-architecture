## Before Starting

- Pull the latest repository changes.
- Read the Architecture documentation.
- Read the Frontend Onboarding document.
- Review the backend API contracts.
- Understand the existing UI structure.
- Preserve the approved architecture.

---

## Constraints

- Do not create APIs.
- Do not modify backend contracts.
- Keep components small and reusable.
- Reuse existing components whenever possible.
- Centralize API communication.
- Use React Router.
- Use React Hook Form.
- Use Tailwind CSS.
- Keep business logic in the backend.
- Keep UI logic readable and maintainable.

## If You Encounter an Architectural Issue

Do not guess.

- Why implementation cannot continue.
